Yes. I would **not** use two full footers stacked everywhere.

I would make **one footer area** with **two modules**:

```markdown
FOOTER SYSTEM

1. Info Footer = global footer
2. Citation Footer = optional citation/provenance module
```

## My recommendation

### Use the **Info Footer** on almost every page.

This is the normal site footer. It holds identity, contact, public links, and the source-line seal.

### Use the **Citation Footer** only on canon / framework / provenance pages.

Do **not** put the full citation footer on every page, because it is heavy. It belongs on pages where source, canon, evidence, citation, or boundary matter directly.

The provenance page should hold the full receipts.

## Final structure

```markdown
ONE FOOTER AREA

Global Info Footer:
On almost all pages.

Citation / Provenance Module:
On canon, framework, tests, provenance, and source-line pages.

Full DOI / DocuSign / copyright record details:
Move to Provenance or Use / Cite / License page.
```

So the answer is:

**One footer system. Two modules.**
Global info always/usually on.
Citation footer only where canon/source/provenance needs to stay visibly attached.
