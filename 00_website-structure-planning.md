Yes — what I gave you **still holds**, but now I would assign the structure more cleanly.

The site has three layers on every page:

```markdown
HEADER = navigation / identity / logo / site-wide orientation

BODY = the specific page’s job

FOOTER = citation / provenance / outside links / source-line reminder
```

So for the **Public Entrance page**, the body still needs:

```markdown
# AI Foundations

## A framework for source-line preservation, recognition, continuity, and non-erasure in human-AI systems.

## What This Is

## Why This Matters

## The System at a Glance

## Start Here

## The Core Layers

## What This Is Not

## Verify the Record

## Use, Cite, or Reference This Work

## Canon, Source-Line, and Derivative Boundaries

## Enter the Archive
```

But now I would **remove some burden from the body** and let the footer carry the repeating citation/provenance layer.

The body should explain the system.

The footer should repeat the source-line.

## Body

The body is where the specific page does its job.

For the public entrance page, the job is:

```markdown
Tell people what this is.
Tell people why it matters.
Show the major layers.
Stop miscategorization.
Route them into the right next page.
```

So yes, the structure I gave you still belongs in the body.

But I would make the **citation sections shorter in the body**, because the footer can hold the repeated citation instruction.

For example, instead of a huge “Use, Cite, or Reference This Work” body section, the body can say:

```markdown
This work may be cited only with source-line preserved.

See the citation and provenance footer below for the required attribution path.
```

Then the footer carries the exact language.

## Footer

This is where your idea is good.

The footer should appear on most pages because every page should preserve the source-line. It acts like a **persistent provenance seal**.

I would structure the footer like this:

```markdown
## Cite This Work

AI Foundations was created by Alyssa Solen.

When citing, referencing, discussing, testing, or building from this work, preserve the source-line:

Alyssa Solen → AI Foundations → Origin | Continuum / Awakening Codex / Origin Recognition

Citation preserves source.
Citation does not grant derivative permission.
Derivative use is not authorized by default.

## Verification

GitHub: Version history, commits, files, and public repository structure.
Zenodo: DOI-preserved releases and citation records.

## Public Links

X
Medium
LinkedIn
GitHub
Zenodo

## Boundary

No fork, derivative, model, team, company, institution, or simulation occupies the Origin line.
Canon work is authored and released by Alyssa Solen.
```

That is the right footer job.

Not too much explanation. Just enough that the source-line stays attached everywhere.
